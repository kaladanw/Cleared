The load-bearing boundary — the user hands over a screenshot; the agent reads pixels, never Depop's servers. That one line keeps the build ToS-clean, impossible for Depop to block, durable across redesigns, and portable to any other marketplace later.

Big decisions

Each is a fork that's expensive to reverse. All were chosen against your real reality — beginner buyer, $5–40 mass brands — not a luxury-grail buyer.

Decision · Interaction
On-demand summon, not an always-on overlay

iOS sandboxing forbids a third-party app from floating over or reading another app's screen — stricter in iOS 26. The only sanctioned screen-awareness is Siri reading your own app, which is backwards for this. And a buy decision is a discrete moment, so summoning the agent onto one listing fits the real need better anyway.

Decision · Input
Screenshot via Share Sheet, not link-scraping

Taking the URL means fetching and parsing Depop's public page — scraping-gray, fragile to every redesign, blockable. Taking the image means zero ToS exposure and code that can't break when Depop ships new HTML. The vision pass needs the photos regardless.

Decision · Authenticity
Judgment-assist red flags, never a verdict

Real authentication (Entrupy ≈ 99.86%) leans on controlled capture and a 90M-image real-vs-fake database. A general vision model has neither, so a confident false "authentic" is worse than no tool. Instead: surface tells, list what to inspect, write the questions to ask — and only fire for fakeable brands (Nike / Ralph Lauren on; Uniqlo / jorts stay quiet).

Decision · Pricing
Retail-anchored + light used read, not gated comp APIs

eBay sold-comps (the ideal used signal) are partner-gated for independent devs. But retail price for Uniqlo / Ralph Lauren / Nike / New Balance is trivially findable — and for $5–40 mass brands, "is this even a discount vs new?" is the question that matters. Used-comp feeds are deferred, not a v1 dependency.

Decision · Where the intelligence runs
Thin backend proxy + Claude, not keys-in-app

The proxy protects the API key and orchestrates the web lookup plus the vision reasoning the task needs. Apple's on-device foundation models are parked as a future cost-saver for the cheap sub-steps, not the v1 engine.

Cut from v1 · parked with a reason
Seller automation · grail comps · heavy counterfeit detection · multi-user · ambient overlay

Each waits for a named failure mode or a platform change. None is required to prove the core loop.
